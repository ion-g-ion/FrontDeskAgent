# Home Assistant Add-on: FrontDeskAgent

FrontDeskAgent combines camera streams, optional event context, and LLM prompts to produce concise front-desk style responses inside Home Assistant.
Responses are generated by live model calls while the add-on is running.

## How to use

Use this add-on when you want camera-aware AI responses that are specific to each view (for example, front door vs driveway) and aligned with your preferred tone.
This is a live model workflow, not a static rules-only pipeline.

Configure your Gemini API key, model, cameras, and prompt sections in the add-on configuration UI.
For larger prompt text, switch to YAML editing mode in the add-on config and use multiline values (`|-`).

## Minimum setup checklist

- Add a valid Gemini API key and model name.
- Configure at least one camera in `cameras`.
- Ensure go2rtc host/ports and `stream_name` are reachable from the add-on.
- Start the add-on and check logs for connection/model errors.

## Exposed Home Assistant entities

After setup, FrontDeskAgent creates entities under the `frontdeskagent` integration/device.

- Per camera `button` (Trigger): Starts a live processing run for that camera (manual test or automation target).
- Per camera `button` (Cancel): Cancels the in-flight run for that camera.
- Per camera `sensor` (Status): Current camera processing state (for example waiting/active), useful in dashboards and automations.
- Global `text` (Status): Home presence/agent mode text value used by the workflow context.
- Global `text` (Notification): Latest generated owner-facing message content.
- Global `todo` (Past Conversations): Persistent history list for prior interactions.
- Global `todo` (Expected Visitors): Persistent list of visitors/packages to expect.

Use these entities as automation triggers/conditions to connect FrontDeskAgent with notifications, lights, locks, or other Home Assistant actions.

## Camera settings fields

Each item in `cameras` represents one camera stream.

- `camera_name`: Friendly camera name used to build the per-camera Home Assistant entity.
- `go2rtc_host`: Hostname or IP address where your go2rtc service runs.
- `go2rtc_api_port`: go2rtc HTTP/WebRTC API port (commonly `1984`).
- `go2rtc_rtsp_port`: go2rtc RTSP port used for stream transport (commonly `8554`).
- `stream_name`: Stream name defined in go2rtc (for example `doorbell`).
- `description`: Natural-language description of what this camera sees; this is passed to the LLM as context.
- `camera_prompt`: Extra camera-specific instructions for the LLM (for example behavior for this view).

## Prompt editing tips

- Home Assistant add-on form inputs for strings are rendered as single-line controls.
- To edit larger prompts comfortably, open the add-on configuration YAML editor.
- Use YAML multiline blocks for prompt values, for example:

```yaml
prompt:
  identity: |-
    You are a polite residential front desk agent.
    Keep responses concise and friendly.
```

## Practical prompt guidance

- Put global behavior in `prompt.identity` (role, tone, brevity).
- Put scene-specific behavior in each camera's `camera_prompt`.
- Keep instructions concrete (what to mention, what to ignore, response length).
